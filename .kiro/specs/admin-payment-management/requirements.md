# Требования: Управление платежами в админ-панели

## Overview

Расширение админ-панели для управления платежами: отображение статусов оплаты, выполнение capture (списание) и refund (возврат), просмотр истории платежей и ошибок.

## Scope

- Добавление колонки "Оплата" в таблицу заказов
- Страница детальной информации о платеже
- Действия: capture (списание), refund (возврат)
- Фильтрация и поиск по статусу оплаты
- История платежей и ошибок

---

## Требования

### 18.1. Отображение статуса оплаты в таблице заказов

**Описание**: В таблице заказов добавить колонку "Оплата" с отображением статуса платежа.

**Детали**:
- Показать статус: `pending` (ожидает), `completed` (оплачен), `failed` (ошибка), `refunded` (возвращён), `partial_refunded` (частичный возврат)
- Для статуса `completed` показать сумму
- Цветовая индикация статусов:
  - `pending` — жёлтый
  - `completed` — зелёный
  - `failed` — красный
  - `refunded` — серый
  - `partial_refunded` — оранжевый

**Критерий приёмки**: В таблице заказов отображается корректный статус оплаты для каждого заказа.

---

### 18.2. Детальная информация о платеже

**Описание**: При клике на статус оплаты открывать детальную информацию о платеже.

**Детали**:
- ID платёжной операции (payment_operation_id)
- Дата создания платежа
- Дата оплаты (если оплачен)
- Сумма платежа
- Валюта (RUB)
- Способ оплаты (SBP)
- Данные плательщика (телефон, email)
- Ссылка на фискальный чек (если есть)
- История статусов

**Критерий приёмки**: Модальное окно показывает полную информацию о платеже.

---

### 18.3. Выполнение capture (списание)

**Описание**: Возможность выполнить списание средств для заказов в статусе `authorized`.

**Детали**:
- Кнопка "Списать" доступна для заказов в статусе `authorized`
- При клике показать подтверждение с суммой
- Поддержка частичного списания (ввод суммы)
- После успешного capture обновить статус заказа
- Показать уведомление об успехе/ошибке

**Критерий приёмки**: Средства списываются, статус заказа обновляется.

---

### 18.4. Выполнение refund (возврат)

**Описание**: Возможность выполнить возврат средств для оплаченных заказов.

**Детали**:
- Кнопка "Вернуть" доступна для заказов в статусе `completed` или `captured`
- При клике показать подтверждение с суммой
- Поддержка полного и частичного возврата
- Проверка: нельзя вернуть больше, чем оплачено
- Проверка: нельзя вернуть платежи старше 90 дней (с предупреждением)
- После успешного refund обновить статус заказа
- Показать уведомление об успехе/ошибке

**Критерий приёмки**: Средства возвращаются, статус заказа обновляется.

---

### 18.5. Фильтрация по статусу оплаты

**Описание**: Добавить фильтр по статусу оплаты в панель заказов.

**Детали**:
- Выпадающий список с статусами: все, ожидает, оплачен, ошибка, возвращён
- Фильтр комбинируется с существующими фильтрами (по статусу заказа, типу доставки)

**Критерий приёмки**: Заказы фильтруются по статусу оплаты.

---

### 18.6. История платежей

**Описание**: Логирование всех действий с платежами.

**Детали**:
- Запись всех вызовов capture, refund
- Хранение: дата, сумма, результат, ошибка (если есть)
- Отображение в детальной информации о платеже

**Критерий приёмки**: История доступна для просмотра.

---

### 18.7. Просмотр ошибок платежей

**Описание**: Отображение истории ошибок при работе с платежами.

**Детали**:
- В детальной информации о платеже показать последние ошибки
- Для каждой ошибки: дата, код ошибки, сообщение
- Возможность повторить операцию после ошибки

**Критерий приёмки**: Ошибки отображаются и понятны администратору.

---

### 18.8. Экспорт платежей

**Описание**: Добавить экспорт платежей в CSV с финансовыми данными.

**Детали**:
- Дополнительные колонки: статус оплаты, сумма оплаты, сумма возврата, дата оплаты
- Фильтры применяются к экспорту

**Критерий приёмки**: CSV содержит финансовую информацию.
