# План реализации: Фискализация (54-ФЗ)

## Overview

Реализация фискализации: интеграция с облачной онлайн-кассой, автоматические чеки при оплате, возврат, email клиенту, админ-панель.

## Задачи

- [x] 1. Расширить конфигурацию FiscalService
  - Добавить переменные окружения для провайдеров
  - Поддержка CloudKassir, МойСклад, Эвотор
  - Добавить FISCAL_INN, FISCAL_NAME, FISCAL_ADDRESS, FISCAL_API_URL, FISCAL_API_KEY
  - _Требования: 19.1_

- [x] 2. Расширить отправку чеков
  - Реализовать формирование чека в формате 54-ФЗ
  - Добавить все обязательные поля (наименование, количество, цена, НДС)
  - Поддержка скидок и итоговой суммы
  - _Требования: 19.2_

- [x] 3. Реализовать чек возврата
  - Метод `sendRefundReceipt(order, refundAmount)`
  - Ссылка на исходный чек
  - Только возвращаемые позиции при частичном возврате
  - _Требования: 19.3_

- [x] 4. Интегрировать отправку email клиенту
  - После успешного чека — отправка на email клиента
  - Использовать ссылку receipt_url
  - Добавить в NotificationService
  - _Требования: 19.4_

- [x] 5. Добавить API эндпоинты
  - POST /api/fiscal/send/:orderId — ручная отправка
  - GET /api/fiscal/status/:orderId — проверка статуса
  - POST /api/fiscal/callback — callback от ПКФ
  - _Требования: 19.5, 19.8_

- [x] 6. Реализовать обработку callback
  - Обновление статуса чека при callback от ПКФ
  - Логирование успешных и неуспешных callback
  - _Требования: 19.8_

- [-] 7. Расширить админ-панель
  - Отображение статуса чека (fiscal_status)
  - Показать receipt_id и receipt_url
  - Кнопка "Повторить отправку" при ошибке
  - CSS стили для статусов чека
  - _Требования: 19.5, 19.6_

- [~] 8. Реализовать обработку ошибок
  - Логирование ошибок в fiscal_error
  - Retry логика (3 попытки с экспоненциальной задержкой)
  - Уведомление админа при критических ошибках
  - _Требования: 19.7_

- [~] 9. Протестировать интеграцию
  - Тестовая отправка чека
  - Проверка статуса
  - Обработка ошибок
  - _Требования: все_

- [~] 10. Контрольная точка
  - Проверить полный цикл: оплата → чек → email
  - Убедиться в корректности данных 54-ФЗ

## Notes

- Использует существующий FiscalService как основу
- Backend: расширение server.js
- Frontend: админ-панель (orders.html)
